## 1. 为什么写代码要用竖屏

### 1.1 竖屏 vs 横屏：一屏多看多少行？

以 27 英寸 4K（3840×2160）显示器为例：

```
横屏：2160 像素高 ÷ 约 24px/行（含行间距）≈ 90 行代码
竖屏：3840 像素高 ÷ 约 24px/行 ≈ 160 行代码

竖屏比横屏多看 70+ 行代码，几乎是翻倍！
```

### 1.2 竖屏的核心优势

```
┌─────────────────────┐
│  IDE 代码区          │
│━━━━━━━━━━━━━━━━━━━━│
│  function foo() {   │
│    // 完整逻辑      │
│  }                  │
│                     │
│  class Bar {        │
│    ...              │  ← 一个屏幕能看到整个类的实现
│  }                  │
│                     │
│  function baz() {   │
│    ...              │
│  }                  │
│━━━━━━━━━━━━━━━━━━━━│
│  终端 / 调试面板     │
└─────────────────────┘
```

| 优势 | 说明 |
|------|------|
| **更多上下文** | 一个函数、一个类、甚至整段逻辑一览无余，不用反复滚屏 |
| **减少分心** | 窄视野更容易聚焦代码，不受两侧多余 UI 干扰 |
| **终端更宽** | 横屏时终端上下很窄，竖屏后终端垂直空间大，看日志、跑命令更爽 |
| **Git Diff 更好用** | 左右对比 diff 时，竖屏的宽高比不适合并排？错了——竖屏 Diff 一般用上下对比，两个窗口上下各占一半，都比横屏大 |
| **文档阅读** | 竖屏看 PDF、Markdown、网页文档更像看 A4 纸，体验天然好 |

### 1.3 什么时候竖屏不合适

```
不适合竖屏的场景：
  ❌ 需要同时看很多横向并列窗口
  ❌ 做 UI 设计/前端开发需要预览宽屏效果
  ❌ 经常看图/看视频（娱乐为主）
  ❌ 电子表格（Excel 更吃宽度）

解决方案：双屏，一横一竖（见第 11 章）
```

## 3. 尺寸选择

### 3.1 竖屏后的实际尺寸感受

```
横屏标称尺寸 → 竖起来后的实际宽高

24 英寸横屏（53.1×29.9cm）→ 竖屏变成 29.9cm 宽 × 53.1cm 高
27 英寸横屏（59.8×33.6cm）→ 竖屏变成 33.6cm 宽 × 59.8cm 高
32 英寸横屏（70.8×39.9cm）→ 竖屏变成 39.9cm 宽 × 70.8cm 高
```

### 3.2 各尺寸竖屏体验

| 尺寸 | 竖屏宽度 | 竖屏高度 | 代码行数（4K） | 适合场景 | 推荐度 |
|------|:------:|:------:|:------------:|------|:----:|
| **24 英寸** | ~30cm | ~53cm | ~120 行 | 桌面紧凑、副屏 | ★★★ |
| **27 英寸** | ~34cm | ~60cm | ~140 行 | **甜点尺寸** | ★★★★★ |
| **28 英寸** | ~35cm | ~63cm | ~145 行 | 27 的略大版 | ★★★★ |
| **32 英寸** | ~40cm | ~71cm | ~160 行 | 桌面够大、单屏为主 | ★★★★ |

### 3.3 尺寸选择原则

```
24 英寸竖屏：
  ✅ 桌面空间有限
  ✅ 做副屏，旁边有横屏主力
  ⚠️ 4K 在 24 英寸上字会偏小，需 150%~175% 缩放
  ⚠️ 2.5K（2560×1440）在 24 英寸竖屏上是最均衡选择

27 英寸竖屏（推荐）：
  ✅ 竖起来高度刚好在舒适视野范围内（不用抬头低头）
  ✅ 4K 搭配 150% 缩放，字体细腻、显示内容多
  ✅ 宽度约 34cm，IDE 左侧文件树 + 右侧代码区刚好不挤
  ✅ 副屏/主屏都合适

32 英寸竖屏：
  ⚠️ 竖起来 71cm 高，需要抬头才能看到顶部
  ⚠️ 建议桌面深度 ≥70cm，否则太近看不过来
  ✅ 如果是 4K 100% 缩放，能看 160+ 行代码，极致效率
  ✅ 适合独立使用（不做双屏中的副屏）
```

## 4. 分辨率选择

### 4.1 竖屏场景下分辨率的意义

分辨率决定了竖屏的**水平像素数**——而竖屏的水平方向就是你代码的**列宽**所在方向：

```
竖屏 4K（3840×2160）→ 竖起来后宽度方向 2160 像素
   150% 缩放 → 等效 1440 像素宽
   IDE 左侧面板 300px + 代码编辑区 1140px ≈ 可以舒适写代码

竖屏 2K（2560×1440）→ 竖起来后宽度方向 1440 像素
   100% 缩放 → 等效 1440 像素宽
   但高度方向只有 2560px ÷ 1 = 2560 像素高

竖屏 1080p（1920×1080）→ 竖起来后宽度方向 1080 像素
   100% 缩放 → 等效 1080 像素（偏窄，IDE 面板 + 代码区挤）
```

### 4.2 分辨率推荐

| 分辨率 | 竖屏后像素 | 推荐缩放 | 等效宽度 | 代码体验 | 推荐度 |
|------|:--------:|:------:|:------:|------|:----:|
| **3840×2160（4K）** | 2160×3840 | 150% | 1440 | 细腻、不累眼、空间够 | ★★★★★ |
| **2560×1440（2K）** | 1440×2560 | 100%~125% | 1152~1440 | 够用，字不如 4K 细腻 | ★★★★ |
| **1920×1200（16:10）** | 1200×1920 | 100% | 1200 | 够用但偏窄 | ★★★ |
| **1920×1080（1080p）** | 1080×1920 | 100% | 1080 | 窄，大段代码体验差 | ★★ |
| **5120×2880（5K）** | 2880×5120 | 200% | 1440 | 极致细腻，贵 | ★★★★ |

### 4.3 关键结论

```
✅ 预算允许直接上 4K → 27 英寸 4K 是甜点中的甜点
✅ 预算有限 2K 也可以 → 24~27 英寸 2K 能省不少钱
❌ 不推荐 1080p → 竖屏后宽度只有 1080px，写代码太窄
✅ 16:10（1920×1200）比 16:9（1920×1080）好 → 多 120 行竖向像素
```

## 5. 面板类型

### 5.1 三种主流面板对比

| 特性 | IPS | VA | TN |
|------|:---:|:---:|:---:|
| 可视角度 | ★★★★★ | ★★★★ | ★★ |
| 色彩准确性 | ★★★★★ | ★★★★ | ★★★ |
| 对比度 | ★★★（1000:1） | ★★★★★（3000:1） | ★★ |
| 响应速度 | ★★★★ | ★★★ | ★★★★★ |
| 黑色表现 | 一般（发灰） | 优秀（真黑） | 差 |
| 价格 | 中等 | 较低 | 低 |
| **写代码推荐** | ✅ **首选** | ✅ 可选 | ❌ 不推荐 |

### 5.2 为什么写代码首选 IPS

```
竖屏场景下 IPS 的优势：
  → 可视角度大：竖屏上下角度差异大（抬头看顶部 vs 平视看中部），
    IPS 保证不同角度看到的颜色和亮度一致
  → 色彩准：代码高亮颜色准确，深色模式下的背景不偏色
  → 文字清晰：IPS 面板文字边缘更锐利

VA 的注意事项：
  → 可视角度比 IPS 略差，竖屏时顶部和底部可能有轻微色偏
  → 对比度高，黑色更深沉，深色主题下体验好
  → 部分 VA 面板有"黑色拖影"问题（深色背景+白色文字滚动时模糊）
    → ⚠️ 对程序员滚动代码影响较大！买 VA 前务必看评测
```

### 5.3 IPS Black 和 Nano IPS

```
IPS Black（如戴尔 U 系列）：
  → 对比度提升到 2000:1（传统 IPS 的两倍）
  → 黑色更深，深色模式下视觉更舒适
  → 价格比普通 IPS 贵 300~500 元

Nano IPS（LG）：
  → 色域更广（DCI-P3 98%+）
  → 色彩更鲜艳
  → 写代码提升不大，设计师更有感
```

## 6. 竖屏专项考量

### 6.1 16:9 vs 16:10 vs 3:2 vs 1:1

竖屏时，**原比例越接近正方形，竖屏体验越好**：

```
原比例  竖屏后比例  竖屏观感
 16:9  →  9:16   →  偏瘦长，24 英寸以下太窄
 16:10 →  10:16  →  好于 16:9，多一截宽度
  3:2  →  2:3    →  接近 A4 纸比例，竖屏黄金比例
  1:1  →  1:1    →  正方形，写代码宽度充裕但高度不够
```

### 6.2 曲面屏能不能竖屏？

```
⭐⭐ 强烈不推荐曲面屏竖屏！
  → 曲面屏设计是按横屏曲率弯的
  → 竖过来等于画面往上下弯曲，视线角度扭曲
  → 长时间使用容易视觉疲劳

例外：极少数高端曲面屏曲率很小（≥3800R），竖屏影响较小，
     但这种情况还不如直接买平面屏。
```

### 6.3 边框宽度

竖屏时两块屏幕拼在一起会更在意边框：

```
边框建议：
  ✅ 窄边框 / 微边框 → 双屏拼接无感
  ✅ 四边等宽 → 竖屏后视觉平衡
  ⚠️ 宽下巴 → 竖屏后变成宽"左下巴"或"右下巴"，不对称
```

## 7. 支架与安装

### 7.1 自带支架：旋转功能是刚需

```
选购时检查自带支架是否支持：
  ✅ 90° 旋转（竖屏）
  ✅ 升降调节（高度舒适）
  ✅ 俯仰调节（角度舒服）
  ⚠️ 左右旋转（双屏时有用）

避坑：
  ❌ 不能旋转的支架 → 竖屏必须额外买支架臂
  ❌ 能旋转但高度固定的 → 竖起来后可能太高或太低
  ❌ 旋转后线材拉扯 → 确认接口位置在侧面而非正下方
```

### 7.2 VESA 接口

买显示器务必确认有 VESA 接口，它让你能脱离原装支架：

```
VESA 规格：
  75×75mm  → 小尺寸（24 英寸以下）常见
  100×100mm → 最主流标准，大多数支架臂都支持

如果没有 VESA：
  → 只能用原厂支架
  → 不能换支架臂
  → 部分品牌有 VESA 转换套件（额外花钱）
```

### 7.3 独立支架臂

```
独立支架臂的优势：
  ✅ 释放桌面空间（原装底座通常占 ∼25cm 深度）
  ✅ 更灵活的调节范围（拉近、推远、转方向）
  ✅ 线材管理（多数支架臂带理线槽）
  ✅ 换显示器不用换支架

推荐品牌：
  北弧（Brateck） → 百元级，性价比高
  NB（North Bayou）→ 经典选择，F80/F100 经典型号
  乐歌（Loctek）  → 产品线全
  爱格升（Ergotron）→ 高端，顺滑如丝，预算 ¥800+
```

### 7.4 竖屏的显示器位置

```
竖屏在双屏中的位置建议：

方案 A（推荐）：横屏正中 + 竖屏在侧
  [竖屏] | [横屏主力]
  适合：横屏做主要工作区，竖屏看文档/终端/代码

方案 B：竖屏正中 + 横屏在侧
  [横屏] | [竖屏主力]
  适合：竖屏写代码为主，横屏看浏览器/设计稿

方案 C：双竖屏
  [竖屏] | [竖屏]
  适合：大量代码+文档并排，纯生产力（少见但效率高）

高度对齐原则：
  → 两块屏幕的物理高度尽量接近
  → 鼠标跨屏移动时不会跳跃太大
```

## 8. 接口与连接

### 8.1 USB-C 一线连（强烈推荐）

插一根线搞定：**视频信号 + 数据传输 + 笔记本充电**。

```
USB-C 显示器的参数要看：

供电功率：
  65W  → 轻薄本够用
  90W  → 主流笔记本都够用
  100W+ → 高性能笔记本也能全速运行

注意：
  ⚠️ 标注"USB-C"但只有 15W → 只能传视频，不能充电
  ⚠️ 标注"USB-C 65W PD"→ 确认你的笔记本支持 PD 充电
```

### 8.2 接口清单

```
必备：
  ✅ HDMI 2.0+ 或 DP 1.4+（至少一个数字接口）
  ✅ USB-C（带供电更好）

加分项：
  ✅ USB Hub（显示器自带 USB 扩展，接键盘鼠标）
  ✅ 3.5mm 音频输出（插耳机方便）
  ✅ 双 HDMI + DP（同时接多台设备）

MacBook 用户特别注意：
  → 优先 USB-C 直连
  → HDMI 走转换器注意 4K@60Hz 需要 HDMI 2.0 以上
  → M1/M2/M3 MacBook Air 只支持 1 台外接显示器
```

## 9. 护眼与长时间使用

程序员每天盯着屏幕 8~12 小时，护眼不是可选项。

### 9.1 护眼功能清单

| 功能 | 作用 | 重要性 |
|------|------|:----:|
| **TÜV 低蓝光认证** | 减少有害蓝光 | ★★★★★ |
| **无频闪（Flicker-Free）** | DC 调光，不闪屏 | ★★★★★ |
| **防眩光雾面屏** | 减少反光 | ★★★★★ |
| **自动亮度调节** | 环境光感应 | ★★★ |
| **低蓝光模式 / 阅读模式** | 一键暖色调 | ★★★★ |

### 9.2 镜面屏 vs 雾面屏

```
✅ 写代码首选雾面屏（防眩光）

镜面屏（亮面）：
  → 色彩更通透
  → 反光严重，窗户/灯光映在屏幕上
  → 长时间使用眼睛更累
  → iMac / Studio Display 是镜面屏代表

雾面屏（磨砂）：
  → 文字边缘微微不如镜面锐利（4K 下几乎不可感知）
  → 几乎不反光
  → 长时间编码舒适
  → 绝大多数办公/编程显示器都是雾面屏
```

### 9.3 亮度和环境光

```
亮度建议：
  白天 → 120~160 cd/m²（约显示器亮度 40%~60%）
  晚上 → 80~100 cd/m²
  ⚠️ 不要在暗室中亮屏（瞳孔反复收缩放大 → 眼疲劳）

环境：
  ✅ 屏幕后方或侧面有柔和光源
  ✅ 屏幕挂灯（如 BenQ ScreenBar）照亮桌面不照屏幕
  ❌ 屏幕背对窗户（白天反光严重）
  ❌ 屏幕正对窗户（亮度差太大）
```

## 11. 双屏与多屏方案

### 11.1 经典方案：一横一竖

```
┌──────────┐  ┌──────────────────────────────┐
│          │  │                              │
│  27 英寸  │  │       27 英寸 横屏             │
│   竖屏    │  │       (主力工作区)              │
│  4K      │  │       4K                      │
│  写代码   │  │                              │
│  文档/终端│  │      浏览器/调试/设计稿         │
│          │  │                              │
└──────────┘  └──────────────────────────────┘
```

优势：竖屏专注写代码看文档，横屏应对需要宽视野的任务。两块同尺寸 4K，高度基本一致，视觉无缝。

### 11.2 笔记本 + 竖屏显示器

```
把笔记本放一侧 → 竖屏显示器做主力

[笔记本屏幕] | [竖屏显示器]
  看文档        写代码
  即时通讯      主力工作区

连接方式：
  → USB-C 一线连（最优）
  → 笔记本放支架上垫高，屏幕高度与显示器对齐
```

### 11.3 超宽屏 + 竖屏

```
竖屏 + 带鱼屏（21:9 / 32:9）

[竖屏 27 4K] | [34 英寸 21:9 带鱼屏]
   代码区           IDE + 浏览器 + 终端 三区并列
```
