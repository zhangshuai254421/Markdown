# 公司电脑远程访问家中实验环境 — 完整方案指南

## 场景说明

| 环境 | 网络状况 | 角色 |
|---|---|---|
| **公司电脑** | 受限网络，只能访问特定白名单网站（通常允许 80/443 端口出站） | 客户端，发起远程连接 |
| **家中电脑** | 完整互联网访问 | 服务端，运行实验环境 |

**核心挑战**：公司防火墙限制了出站流量，传统的直连方案（端口映射 + 公网 IP）大概率不可行。

---

## 方案总览与对比

| 方案 | 难度 | 费用 | 依赖条件 | 成功率 | 推荐指数 |
|---|---|---|---|---|---|
| **Cloudflare Tunnel** | ⭐ 低 | 免费 | 需一个域名 | 🔥 最高 | ⭐⭐⭐⭐⭐ |
| **frp 内网穿透** | ⭐⭐ 中 | ~¥50/月(VPS) | 需一台云服务器 | 很高 | ⭐⭐⭐⭐ |
| **Tailscale** | ⭐ 低 | 免费 | 公司允许安装软件 | 中等 | ⭐⭐⭐ |
| **Chrome 远程桌面** | ⭐ 低 | 免费 | Google 服务在白名单内 | 中高 | ⭐⭐⭐ |
| **SSH 反向隧道** | ⭐⭐ 中 | ~¥50/月(VPS) | SSH(22端口)未被封 | 中等 | ⭐⭐⭐ |
| **ZeroTier** | ⭐ 低 | 免费 | 公司允许安装软件 | 中低 | ⭐⭐ |

---

## 方案一：Cloudflare Tunnel（最推荐）

### 原理

```
┌──────────────┐        HTTPS(443)         ┌──────────────┐        ┌──────────────┐
│  公司电脑     │ ◄──────────────────────► │  Cloudflare   │ ◄─────► │  家中电脑     │
│  (浏览器)     │    cloudflared 隧道        │  Edge 网络    │ 出站连接 │  (服务端)     │
└──────────────┘                            └──────────────┘        └──────────────┘
```

- 家中电脑主动向 Cloudflare 边缘网络建立**出站** HTTPS 连接（这个连接几乎不会被任何防火墙拦截）
- 你在公司通过 Cloudflare 分配的域名 + 浏览器即可访问家中服务
- **不需要家里有公网 IP**，**不需要在公司防火墙上打洞**

### 为什么它能穿透公司防火墙

公司防火墙通常采用"白名单"策略，但 **HTTPS (443端口) 出站几乎永远开放**，否则连百度、腾讯、阿里云等常用网站都无法访问。Cloudflare Tunnel 正是利用了这一点。

### 操作步骤

#### 第 1 步：准备一个域名

1. 在任意域名注册商购买一个域名（如 Namesilo、GoDaddy、阿里云万网），最便宜的约 ¥10/年
2. 将域名的 DNS 服务器指向 Cloudflare：
   - 在 Cloudflare 官网注册账号
   - 添加你的站点（Add Site）
   - 按指引修改域名注册商处的 DNS 服务器为 Cloudflare 分配的地址
   - 等待 DNS 生效（通常几分钟到几小时）

#### 第 2 步：在家中电脑安装 cloudflared

**Windows（家中电脑）**：
```powershell
# 方式一：下载安装包
# 访问 https://github.com/cloudflare/cloudflared/releases
# 下载 cloudflared-windows-amd64.msi 或 .exe

# 方式二：使用 winget
winget install --id Cloudflare.cloudflared
```

**Linux（家中电脑）**：
```bash
# Debian/Ubuntu
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb -o cloudflared.deb
sudo dpkg -i cloudflared.deb

# 或使用一键脚本
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 -o cloudflared
chmod +x cloudflared
sudo mv cloudflared /usr/local/bin/
```

#### 第 3 步：认证 cloudflared

```bash
cloudflared tunnel login
```

执行后会打开浏览器，登录你的 Cloudflare 账号并授权域名。完成后证书文件保存在 `~/.cloudflared/cert.pem`。

#### 第 4 步：创建隧道

```bash
# 创建一条隧道（取个名字，比如 home-lab）
cloudflared tunnel create home-lab

# 输出会显示隧道 ID 和凭据文件路径
# 凭据文件保存在 ~/.cloudflared/<Tunnel-UUID>.json
```

#### 第 5 步：配置隧道

创建配置文件 `~/.cloudflared/config.yml`：

```yaml
tunnel: <上一步生成的 Tunnel-UUID>
credentials-file: C:\Users\<你的用户名>\.cloudflared\<Tunnel-UUID>.json

ingress:
  # Windows 远程桌面 (RDP)
  - hostname: rdp.yourdomain.com
    service: rdp://localhost:3389

  # SSH 访问
  - hostname: ssh.yourdomain.com
    service: ssh://localhost:22

  # 家中电脑上的 Web 服务（如 Jupyter、VS Code Server、Portainer 等）
  - hostname: lab.yourdomain.com
    service: http://localhost:8080

  # VS Code Server (code-server)
  - hostname: code.yourdomain.com
    service: http://localhost:8443

  # 兜底规则：其他请求返回 404
  - service: http_status:404
```

> **提示**：每增加一个子域名，都需要在 Cloudflare DNS 控制台添加对应的 CNAME 记录，指向 `<Tunnel-UUID>.cfargotunnel.com`。Cloudflare 也支持通过 `cloudflared tunnel route dns` 命令自动添加。

#### 第 6 步：添加 DNS 路由

```bash
# 为每个子域名添加 DNS 记录
cloudflared tunnel route dns home-lab rdp.yourdomain.com
cloudflared tunnel route dns home-lab ssh.yourdomain.com
cloudflared tunnel route dns home-lab lab.yourdomain.com
cloudflared tunnel route dns home-lab code.yourdomain.com
```

#### 第 7 步：安装为系统服务（开机自启）

```bash
# Windows/Linux 通用
cloudflared service install

# 启动服务
cloudflared service start

# 查看状态
cloudflared service status
```

#### 第 8 步：在公司电脑上访问

1. 打开浏览器
2. 访问 `https://lab.yourdomain.com` → 家中 Web 服务
3. 访问 `https://code.yourdomain.com` → 家中 VS Code Server
4. SSH：使用支持 HTTP 代理的 SSH 客户端，或直接通过浏览器终端访问

## 方案二：frp 内网穿透（需一台 VPS）

### 原理

```
┌──────────────┐         ┌──────────────────┐         ┌──────────────┐
│  公司电脑     │ ──────► │  云服务器 (VPS)   │ ◄────── │  家中电脑     │
│  (客户端)     │  任意端口 │  有公网 IP        │  frpc   │  (frp 客户端) │
└──────────────┘         └──────────────────┘         └──────────────┘
```

### 为什么推荐

- VPS 有**固定公网 IP**，公司只需访问这个 IP，不走任何第三方穿透服务
- 协议灵活：TCP/UDP/HTTP/HTTPS 都支持
- 如果公司允许访问特定外部 IP + 端口，这个方案比 Cloudflare Tunnel 更可控

### 操作步骤

#### 第 1 步：购买云服务器

推荐：
| 厂商 | 最低配置 | 参考价格 |
|---|---|---|
| 阿里云 ECS | 1核1G，1M带宽 | ~¥50/月 |
| 腾讯云 CVM | 1核1G，1M带宽 | ~¥50/月 |
| 华为云 | 1核1G，1M带宽 | ~¥50/月 |
| Vultr / DigitalOcean | 1核1G，1TB流量 | $6/月 |

> 关键要求：有公网 IPv4 地址。建议选国内厂商（延迟低），操作系统用 Ubuntu 22.04。

#### 第 2 步：在 VPS 上部署 frp 服务端 (frps)

```bash
# SSH 登录 VPS
ssh root@<你的VPS公网IP>

# 下载 frp（替换为最新版本号）
cd /opt
wget https://github.com/fatedier/frp/releases/download/v0.58.1/frp_0.58.1_linux_amd64.tar.gz
tar -xzf frp_0.58.1_linux_amd64.tar.gz
mv frp_0.58.1_linux_amd64 frp
cd frp
```

创建 `/opt/frp/frps.toml`：
```toml
bindPort = 7000

# 认证令牌（务必修改为强密码！）
auth.token = "your-strong-token-here-至少32位随机字符串"

# 服务端 Dashboard（可选）
webServer.addr = "0.0.0.0"
webServer.port = 7500
webServer.user = "admin"
webServer.password = "admin123"
```

创建 systemd 服务 `/etc/systemd/system/frps.service`：
```ini
[Unit]
Description=frp server
After=network.target

[Service]
Type=simple
ExecStart=/opt/frp/frps -c /opt/frp/frps.toml
Restart=on-failure
RestartSec=5s

[Install]
WantedBy=multi-user.target
```

```bash
systemctl daemon-reload
systemctl enable frps
systemctl start frps
systemctl status frps   # 确认运行正常
```

#### 第 3 步：开放 VPS 防火墙端口

```bash
# 如果是 ufw 防火墙
ufw allow 7000/tcp    # frp 控制端口
ufw allow 3389/tcp    # RDP
ufw allow 2222/tcp    # SSH（转发用）
ufw allow 8080/tcp    # Web 服务

# 同时需要在云厂商控制台的安全组中开放这些端口！
```

#### 第 4 步：在家中电脑配置 frp 客户端 (frpc)

下载 Windows 版 frp：https://github.com/fatedier/frp/releases

创建 `frpc.toml`：
```toml
# 连接到 VPS 服务端
serverAddr = "<VPS公网IP>"
serverPort = 7000
auth.token = "your-strong-token-here"

# 转发 RDP（远程桌面）
[[proxies]]
name = "rdp"
type = "tcp"
localIP = "127.0.0.1"
localPort = 3389
remotePort = 3389

# 转发 SSH（如果家中电脑有 SSH 服务）
[[proxies]]
name = "ssh"
type = "tcp"
localIP = "127.0.0.1"
localPort = 22
remotePort = 2222

# 转发 Web 服务（如 Jupyter、VS Code Server）
[[proxies]]
name = "web"
type = "tcp"
localIP = "127.0.0.1"
localPort = 8080
remotePort = 8080

# 转发 VS Code Server
[[proxies]]
name = "code-server"
type = "tcp"
localIP = "127.0.0.1"
localPort = 8443
remotePort = 8443
```

启动 frpc：
```powershell
# 在 frp 目录下
.\frpc.exe -c .\frpc.toml
```

#### 第 5 步：在 VPS 上为 RDP 配置 TCP 代理

RDP 协议不能直接走 HTTP 反向代理，但 frp 的 TCP 模式**原生支持**，无需额外配置。公司电脑直接通过 `VPS公网IP:3389` 即可远程桌面。

#### 第 6 步：在公司电脑上使用

| 服务 | 访问地址 |
|---|---|
| RDP 远程桌面 | `<VPS公网IP>:3389` |
| SSH | `ssh -p 2222 <VPS公网IP>` |
| Web 服务 | `http://<VPS公网IP>:8080` |

## 方案三：Tailscale（最简单但依赖安装）

### 原理

Tailscale 基于 WireGuard 协议，创建一个私有 Mesh 网络。它会自动尝试各种 NAT 穿透技术（UPnP、NAT-PMP、STUN、DERP 中继），在大多数网络环境下能直连成功。

### 前提条件

- 公司电脑**允许安装 Tailscale 客户端**（不需要管理员权限）
- 公司防火墙没有深度包检测（DPI）拦截 WireGuard 流量

### 操作步骤

#### 第 1 步：注册 Tailscale

访问 https://login.tailscale.com ，使用 Google/GitHub/Microsoft 账号登录（免费版支持最多 3 个用户、100 台设备）。

#### 第 2 步：在家中电脑安装 Tailscale

```powershell
# Windows
winget install tailscale

# 或从官网下载：https://tailscale.com/download
```

安装后登录：
```powershell
tailscale up
# 浏览器会自动打开完成认证
```

#### 第 3 步：在公司电脑安装 Tailscale

同第 2 步。

#### 第 4 步：验证连接

```powershell
# 查看网络中的设备
tailscale status

# 输出示例：
# 100.x.x.x  home-pc    zhangshuai@  windows  -
# 100.x.x.y  work-pc    zhangshuai@  windows  active; direct
```

#### 第 5 步：使用

| 服务 | 访问地址 |
|---|---|
| RDP | `100.x.x.x` (家中电脑的 Tailscale IP) |
| SSH | `ssh user@100.x.x.x` |
| Web 服务 | `http://100.x.x.x:8080` |

> **优点**：配置极简，Tailscale 分配的 100.x.x.x IP 在公司电脑上可以直接 ping 通。
> **缺点**：如果公司防火墙使用 DPI 且识别出 WireGuard 协议特征并拦截，则无法使用。

## 方案四：Chrome 远程桌面（零配置备选）

### 前提条件

- 公司浏览器能访问 Google 服务（accounts.google.com、remotedesktop.google.com）
- 家中电脑安装了 Chrome 浏览器

### 操作步骤

#### 第 1 步：在家中电脑设置

1. 打开 Chrome，访问 https://remotedesktop.google.com/access
2. 点击"设置远程访问"（Set up remote access）
3. 下载 Chrome Remote Desktop Host 组件
4. 设置一个 PIN 码（至少 6 位数字）

#### 第 2 步：在公司电脑访问

1. 打开 Chrome，访问 https://remotedesktop.google.com/access
2. 登录同一个 Google 账号
3. 点击家中电脑名称，输入 PIN 码即可远程桌面

> **优点**：完全免费，零配置，不需要域名/VPS。
> **缺点**：仅限远程桌面，无法直接访问家中其他 Web 服务；依赖 Google 服务可达。

## 方案五：SSH 反向隧道（技术向方案）

### 适用场景

公司防火墙允许 22 端口出站（或你可以通过 HTTP 代理转发 SSH）。

### 操作步骤

#### 第 1 步：购买 VPS（同方案二）

#### 第 2 步：从家中电脑建立反向 SSH 隧道

```bash
# 在家中电脑执行（Linux/WSL/Git Bash）
# 将 VPS 的 3389 端口转发到本地的 3389 (RDP)
ssh -R 3389:localhost:3389 root@<VPS_IP> -N -o ServerAliveInterval=60

# 同时转发多个端口
ssh -R 3389:localhost:3389 \
    -R 8080:localhost:8080 \
    -R 2222:localhost:22 \
    root@<VPS_IP> -N -o ServerAliveInterval=60
```

#### 第 3 步：在 VPS 上开放转发端口

```bash
# 编辑 SSH 配置，允许远程主机绑定端口
vi /etc/ssh/sshd_config
# 添加：
GatewayPorts yes

systemctl restart sshd
```

#### 第 4 步：使用 autossh 保持隧道（防止断开）

```bash
# 安装 autossh
sudo apt install autossh   # Linux
# Windows 可在 WSL 中执行

# 创建持久隧道
autossh -M 0 \
    -R 3389:localhost:3389 \
    -R 8080:localhost:8080 \
    root@<VPS_IP> -N -o ServerAliveInterval=60
```

#### 第 5 步：在公司电脑访问

同方案二：直接访问 `<VPS_IP>:3389` 等端口。

## 方案六：ZeroTier（Tailscale 替代方案）

如果 Tailscale 被封，可以尝试 ZeroTier（协议不同，可能穿透率不同）。

### 操作步骤

1. 在 https://my.zerotier.com 注册，创建一个 Network，获取 Network ID
2. 家中电脑：安装 ZeroTier → Join Network → 输入 Network ID
3. 公司电脑：同步骤 2
4. 在 ZeroTier 管理页面授权两台设备
5. 使用 ZeroTier 分配的 IP（通常是 10.x.x.x 或 172.x.x.x）直接访问

## 进阶技巧

### 1. 在家中搭建 code-server（浏览器版 VS Code）

```bash
# 安装 code-server
curl -fsSL https://code-server.dev/install.sh | sh

# 启动（绑定到本地）
code-server --bind-addr 127.0.0.1:8443 --auth password
```

然后通过 Cloudflare Tunnel 或 frp 暴露 `127.0.0.1:8443`，在公司浏览器中就能使用完整 VS Code。

### 2. 搭建 Jupyter Lab（Python 开发）

```bash
pip install jupyterlab
jupyter lab --ip 127.0.0.1 --port 8888 --no-browser
```

然后通过隧道暴露端口 8888。

### 3. 搭建 Portainer（Docker 管理面板）

```bash
docker run -d -p 9000:9000 --name portainer \
  --restart=always \
  -v /var/run/docker.sock:/var/run/docker.sock \
  portainer/portainer-ce
```

通过隧道暴露端口 9000，在公司浏览器中管理家中 Docker 容器。

### 4. 使用 Cloudflare Access 增加安全层

如果选择方案一（Cloudflare Tunnel），可以配合 Cloudflare Access 实现**零信任访问**：

1. 在 Cloudflare Zero Trust 控制台 → Access → Applications
2. 添加你的子域名
3. 配置认证策略（如 Google 账号登录、邮箱验证码等）
4. 只有通过认证的用户才能访问你的服务

这样即使域名被扫描到，未经认证的请求也无法访问。

### 5. 配置家中电脑 Wake-on-LAN

如果不想让家中电脑 7×24 小时开机，可以配合 WOL 使用：

1. 在家中局域网放置一个低功耗设备（如树莓派）作为跳板机，24 小时运行隧道客户端
2. 树莓派上配置 WOL：
   ```bash
   sudo apt install etherwake
   # 唤醒家中主机（需主板支持且 BIOS 中开启 WOL）
   sudo etherwake -i eth0 <家中电脑MAC地址>
   ```
3. 通过隧道登录树莓派 → 执行 WOL → 等待家中电脑启动 → 再连接

## 故障排查清单

### Cloudflare Tunnel

| 问题 | 排查命令 / 方法 |
|---|---|
| 隧道连接不上 | `cloudflared tunnel info home-lab` |
| 域名解析不对 | `nslookup rdp.yourdomain.com` |
| 403 错误 | 检查 ingress 配置中的 hostname 是否匹配 |

### frp

| 问题 | 排查方法 |
|---|---|
| frpc 连不上 frps | 检查 VPS 安全组是否开放 `bindPort`；`telnet <VPS_IP> 7000` |
| 端口不通 | 检查 VPS 安全组和本地防火墙；检查 `remotePort` 是否被占用 |
| 频繁断开 | 增加 VPS 带宽；降低转发的服务数量 |

### 通用

```bash
# 测试公司到 VPS 某端口是否通（PowerShell）
Test-NetConnection -ComputerName <VPS_IP> -Port 443

# 追踪路由
tracert <目标IP或域名>
```

## 推荐方案组合

根据你的场景（公司电脑只能访问特定网站），推荐的组合策略：

```
第一优先级：Cloudflare Tunnel + code-server/JupyterLab
    ↓ 如果失败（极少见，因为 HTTPS 几乎不会被封）
第二优先级：frp VPS 穿透
    ↓ 如果公司连特定外部 IP 也限制
第三优先级：Chrome 远程桌面（纯远程桌面方案）
```

**最佳实践**：
1. 在家中电脑安装 **code-server**（浏览器版 VS Code）或 **Jupyter Lab**
2. 使用 **Cloudflare Tunnel** 暴露到公网
3. 在公司浏览器中直接编码、运行实验
4. 需要的只是一个浏览器，无需安装任何额外软件
